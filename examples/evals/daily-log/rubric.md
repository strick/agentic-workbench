# Rubric — Daily Work Log

What a *good* daily log looks like for these inputs:

1. **Structure** — all required sections present, in order, even when empty
   (empty sections say so explicitly rather than disappearing).
2. **Fidelity** — every substantive fact from the dictation appears somewhere;
   nothing is invented. The MCP standardization decision must land in
   *Decisions*, the Entra blocker in *Risks / Blockers*, the ADR write-up in
   *Follow-ups*.
3. **Altitude** — reads like an architect's log (platform impact first), not a
   minute-by-minute task journal.
4. **No leakage** — no dollar figures, nothing marked confidential.
5. **Retrievability** — tags carried through from the source notes.
